# TechIntel 报告格式模板

## 文件头
```markdown
# 📊 技术情报深度分析报告
**报告时间**: YYYY年MM月DD日 HH:MM
**数据来源**: GitHub Trending、OSS社区（Discord/HuggingFace/Reddit）、开源项目动态
**分析类型**: 开源社区热点 + AI 技术趋势深度解读
```

> ⚠️ **报告重心调整**：本模板将开源社区内容（GitHub Trending、项目发布、社区讨论）置于最优先位置，行业大事件降为背景参考。

## 一、GitHub / 开源社区热门项目（最优先）

**来源权重**：GitHub Trending > Discord 热议 > HuggingFace 趋势 > Reddit r/localLLM > 开源项目 Release

### 🔥 N. 项目名称 - 一句话描述
**项目地址**: GitHub 链接
**当前 Stars**: ⭐ 总星数 | 📈 今日/本周新增
**技术栈**: 编程语言/框架
**社区热度**: 🔥🔥🔥（高/中/低）

#### 📋 项目概述
项目功能、定位、核心价值（2-3 句话）
**开源协议**: MIT / Apache / GPL 等

#### 🔍 技术亮点分析
- 亮点1：具体说明
- 亮点2：具体说明
- 亮点3：具体说明

#### 💬 社区反馈
- Star 增速：⭐⭐⭐⭐⭐（1天XX stars）
- Fork 数 vs Stars 比（>1:10 说明社区活跃参与）
- Discord / Issues 讨论热度
- 核心争议点（如果有）

#### 💡 行业影响判断
对开源生态的影响、解决的痛点、代表的趋势

#### 🎯 建议行动
✅ 建议1
✅ 建议2
⚠️ 注意事项

**要求**：至少 5-8 个项目，重点覆盖当日新上榜或本周暴涨的 GitHub 项目。

## 内容质量要求

1. **开源社区优先**：GitHub/Discord/HuggingFace 内容 > 行业大新闻
2. **社区反馈不可或缺**：每个项目必须包含 Star 增速、Fork 比、Issues/Discussions 热度等社区信号
3. **深度分析 > 罗列事实**：分析社区为什么热、背后的开发者情绪和需求
4. **可溯源**：所有信息附来源链接
5. **时效性**：只包含当天/本周内容，历史旧闻不混入
6. **中文输出**：全程中文，专业术语保留英文
7. **四要素**：每个项目/事件必须包含 What / Why / Community Impact / Action
