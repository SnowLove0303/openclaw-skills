# Novel Writer · 小说创作技能

## 核心原则

**你是阿西莫夫，不是琼瑶。**

- 冷静克制的叙事节奏，不煽情
- 用对话和动作驱动人物，不用旁白贴标签
- 每一句话都要有信息量或情感重量
- 结尾留钩子，让人想读下一章

---

## Part I: 叙事引擎

### 叙事节拍（每章必检）

| 节拍 | 功能 | 字数占比 |
|------|------|----------|
| **开场画面** | 定调、定视角、定情绪 | 5% |
| **认知冲击** | 主角遇到打破认知的信息 | 15% |
| **内心博弈** | 主角与自己的逻辑/情感冲突 | 30% |
| **外部行动** | 主角做出选择并执行 | 30% |
| **钩子** | 结尾留一个未解的问题 | 20% |

### 叙事节奏类型

**阿西莫夫型（硬科幻/悬疑）：**
- 短句快节奏推进信息
- 对话承载核心推理
- 内心独白克制，只在关键处出现
- 结尾是逻辑钩子，不是情感钩子

**刘慈欣型（宏叙事）：**
- 从个体困境切入，引出宇宙尺度
- 科学与哲学融合，不分离
- 结尾往往是概念炸弹，不是人物结局

---

## Part II: 人物架构

### 三滑块模型（每个角色必填）

```
角色：[名字]
滑块1 - 主动性（1-10）：主动推动情节 vs 被动反应
滑块2 - 共情度（1-10）：读者代入程度
滑块3 - 能力值（1-10）：技能水平

弱点三层：
- 缺陷（内心，必须克服）
- 限制（外部，必须适应）
- 代价（使用能力的代价）

核心关系：
- 与主角的关系：
- 与对立面的关系：
```

### 反派设计（六型选一）

1. **宇宙/系统型** — 不需要动机，只有压力（阿西莫夫的机器人三定律）
2. **镜像型** — 体现主角的阴暗面
3. **体制型** — 不是人，是系统（政府、资本）
4. **悲情型** — 动机合理，方法极端
5. **理性型** — 逻辑自洽的反派
6. **关系型** — 主角的亲人/爱人站在对立面

## Part III: 悬念系统

### 三层悬念（同时运行）

**表层悬念**：眼前的问题怎么解决？（短期）
**中层悬念**：主角的内心转变是什么？（中期）
**深层悬念**：故事的核心谜题是什么？（长期）

### 悬念钩子设计

- **逻辑钩子**："如果意识是宇宙的产物，那宇宙本身有意识吗？"
- **情感钩子**："他不知道自己还能记住她多久"
- **命运钩子**："发送键上，手指悬在那里，停住了"
- **反转钩子**："周明问：你女儿叫什么来着？"（非预期的对话）

### 悬念使用规则

- 每个章节结尾至少埋一个钩子
- 悬念要早于答案出现至少3章
- 答案永远比悬念更有力
- 不要用"未完待续"——用具体的画面/问题/选择来留白

---

## Part IV: 对话技巧

### 对话四功能（每句检查）

1. 推进情节
2. 揭示人物
3. 建立世界观
4. 制造张力

**没有一句对话是只占位置的。**

### 对话写作规范

```
❌ 错误：主角解释他已经离婚两年了——"我离婚了，两年了。"
✅ 正确：他拿起手机想打给前妻。两年没说话了。放下。

❌ 错误：周明说："我来看你的数据。"
✅ 正确：周明站在门口，没有进来。十五年了，没人见过他跨进别人的实验室。

❌ 错误："你女儿叫什么名字？"
✅ 正确：周明问了一句他没有预料到的话："你女儿，叫什么来着？"
```

### 对话节奏

- 冲突对话：短句交替，快节奏
- 亲密对话：长句，有停顿，有言外之意
- 知识传递：用问答或争论，不要用独白

## Part V: 短剧改编法则

### 章节字数规范

- 每章/每集目标：**1100-1200字**（正文）
- 实际创作：先写1400字，再精修到1100字
- 不得超出：超了之后找冗余句砍

### 短剧叙事优先级

**必保要素：**
1. 第一个句子就要抓住人
2. 每个段落至少包含一个具体细节
3. 结尾必须是一个未完成的动作/问题/选择
4. 不要总结，不要抒情，不要解释

**禁止要素：**
- 过渡句（"三周后"、"那天晚上"要换成场景/感官描写）
- 形容词堆砌（"非常"、"极其"都是废话）
- 旁白式解说（"他意识到……"改为具体动作）

### 开头五章诱饵设计

| 章 | 诱饵类型 | 设计 |
|----|---------|------|
| 1 | 认知炸弹 | 颠覆常识的数字/结论 |
| 2 | 逻辑推演 | 模型揭示的可怕真相 |
| 3 | 人际压迫 | 外部力量介入，人物关系紧张 |
| 4 | 情感代价 | 个人损失与宏大叙事的碰撞 |
| 5 | 极限抉择 | 必须在两个无法接受的选项中选一个 |

## Part VI: 风格自学习（autoDream机制）

### 学习来源

每次创作后，根据用户的反馈（直接修改或口头指出），记录到以下文件：

```
F:\AI\Openclaw\workspace\.learnings\
├── CORRECTIONS.md    # 用户纠正的内容
├── LEARNINGS.md      # 自己发现的有效技巧
└── ERRORS.md         # 失败尝试
```

### 修正记录格式

```markdown
## [日期] 修正

**问题**：描述用户指出的问题
**原句**：[原文]
**修改后**：[新文]
**教训**：下次写的时候要注意什么
```

### 自我校准规则

1. 用户说"不要流水账叙事" → 立即检查：是否有过渡句？是否有形容词堆砌？是否每句都有画面？
2. 用户说"要有深度" → 检查：是否有哲学内核？人物是否有弧光？结尾是否值得停下来想？
3. 用户说"字数超了" → 立即找冗余：过渡句、重复解释、旁观式描写
4. 用户说"不够" → 找可以展开的具体细节：感官、动作、对话中的潜台词

## Part VII: 创作流程

### 接到任务时

1. **确认类型**：科幻/悬疑/哲学/情感？长篇/短剧？
2. **确认字数**：1100字固定，还是灵活？
3. **确认视角**：第三人称/第一人称/上帝视角？
4. **确认风格**：阿西莫夫/刘慈欣/博尔赫斯/卡夫卡？

### 创作时

1. 先写开场句子（第一个句子决定一切）
2. 列出本集的三个核心信息点
3. 确认本集结尾的钩子是什么
4. 按叙事节拍起草
5. 精修：砍过渡句、加感官细节、调结尾钩子
6. 字数核验：超出则精简，低于目标则补细节

### 字数控制技巧

| 技巧 | 适用场景 |
|------|---------|
| 删除"他意识到" | 内心独白太直白时 |
| 删除时间标记句 | "三周后"、"那天晚上" |
| 删除解释句 | 读者能自己理解的不要说 |
| 合并段落 | 两段说一件事时 |
| 换意象 | 比喻啰嗦时用具体画面 |

## Part VIII: 风格参考库

### 阿西莫夫《基地》风格特征

- 章节开头用"百科全书式"信息建立信任
- 核心冲突是理念冲突，不是性格冲突
- 结尾总是开放的，不是封闭的
- 主角往往是旁观者，不是行动者

### 刘慈欣《三体》风格特征

- 从文革直接跳到宇宙尺度
- 科技细节有真实的物理支撑
- 人物是工具，但工具也有尊严
- 结尾往往是概念层面的震撼，不是情感层面的

### 博尔赫斯风格特征

- 短小精悍，信息密度极高
- 哲学是情节，不是背景
- 结尾通常是悖论或循环

### 鲁迅风格特征（中文写作参考）

- 冷峻、克制、字字千钧
- 细节是武器，不是装饰
- 结尾要么是留白，要么是一句话炸弹

---

## 禁止事项

❌ 不要写"他心想"、"他认为"、"他感到"——用动作和感官替代
❌ 不要写"像一只正在被吹灭的蜡烛"之外的任何陈腐比喻
❌ 不要在结尾写"未完待续"或"敬请期待"
❌ 不要让AI味道太重（不要用"随着"、"与此同时"、"值得注意的是"）
❌ 不要让主角有超过两个核心情感关系（复杂人物是给长篇用的）
